# Высокоточные вычисления (GMP/MPFR)

## Цель работы

Сравнить результаты вычисления функции с использованием:
- стандартных типов данных (`double`, `long double`)
- библиотеки GMP (`mpz_t`, `mpf_t`, `mpq_t`)
- библиотека MPFR

## Задание

Вычислить значение функции:

```
f(x, y) = (1682·x·y⁴ + 3·x³ + 29·x·y² − 2·x⁵ + 832) / 107751
```

при `x = 192119201` и `y = 35675640`.

**Ожидаемый результат: 1783**

|-----|-----------|------------|

---

## Почему стандартные типы дают неверный результат

Промежуточное значение `x⁵` при `x = 192119201`:

```
x⁵ ≈ 2.56 × 10⁴² — это 42 значащих цифры
```

| Тип | Точность (бит) | Значащих цифр | Достаточно? |
|-----|---------------|---------------|-------------|
| `double` | 64 | ~15 | ❌ |
| `long double` | 80 | ~18 | ❌ |
| `mpz_t` / `mpf_t` | произвольная | неограничено | ✅ |

## Требования

- Компилятор `gcc` / `g++` с поддержкой C99 и C++17
- Библиотека [GMP](https://gmplib.org/) версии 6.x и выше

### Установка GMP (Linux)
```bash
sudo apt install libgmp-dev
```

### Установка GMP (macOS)
```bash
brew install gmp
```

---

## Компиляция и запуск

### C
```bash
gcc main_c.c -o main_c -lgmp
./main_c
```

### C++ (сравнение типов)
```bash
g++ main_cpp_compare.cpp -o main_cpp_compare -lgmp
./main_cpp_compare
```

### C++ (только GMP mpf_t)
```bash
g++ main.cpp -o main_cpp -lgmp
./main_cpp
```

## Вывод

Стандартные типы `double` и `long double` не обеспечивают достаточной точности
для вычислений с большими числами, так как их мантисса ограничена.
Библиотека GMP позволяет работать с числами **произвольной точности**,
что гарантирует верный результат независимо от размера чисел.
